BVN Aizen Step22-24 Final Audit / Cleanup
================================================

Scope
-----
This package is based on Step19-21 and adds the final Aizen-only cleanup requested by the user:
1) Step22: character action-state/input responsiveness audit.
2) Step23: sound / voice / effect trigger mapping audit.
3) Step24: final Aizen timeline raw-call coverage audit.
4) Extra requested feature: add the UI hint “按“p”补满怒气” under the battle HP area and implement P to fill both P1/P2 qi.
5) Extra requested tuning: skill-effect vertical height alignment based on original SWF timeline semantics.
6) Extra requested tuning: reduce input-to-action delay using a short input buffer.

Added UI / Debug Function
-------------------------
- Added a small centered label under the battle HP area:
  按“p”补满怒气
- Press P during battle:
  P1.qi = P1.qi_max
  P2.qi = P2.qi_max
- It does not change HP, energy, fzqi, score, KO state, or defense state.

Input Responsiveness
--------------------
- Added 0.12 second input buffer for:
  J / U / I / K / L
- Purpose:
  If the player presses a button a few frames before the legal cancel/action window, the action is executed on the first valid frame instead of being lost.
- This is closer to Flash per-frame polling behavior and makes combos/skill chaining feel more immediate.
- No action startup frames were deleted; the original action timelines remain intact.

Effect Height Alignment
-----------------------
- bsmc vertical anchor:
  currentTarget.y - 25
  Source: addAttacker("bsmc", {y:{followTarget:true,offset:-25}, applyG:false})
- zh3mc vertical anchor:
  caster fighter origin y at creation time
  Source: addAttacker("zh3mc", {applyG:false})
- bisha XG vertical anchor:
  caster.y - 50, converted through world_to_screen before HUD-layer playback
  Source: EffectCtrler.bisha(target.x, target.y - 50)
- These are encoded in BattlePlaceholder and noted in aizen_effect_manifest metadata.
